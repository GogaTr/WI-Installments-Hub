# WI Installments Hub

**WI Installments Hub** არის WordPress / WooCommerce პლაგინი, რომელიც საშუალებას გაძლევთ მართოთ განვადების განაცხადები პირდაპირ ადმინისტრატორის პანელიდან.

პლაგინი შექმნილია იმ ბიზნესებისთვის, რომლებიც განვადების პროცესს მართავენ ოპერაციულად — არა მომხმარებლის checkout-იდან, არამედ შიდა გუნდით.(ხელით განვადების ინიციალიცაზიისთვის)

---

## მხარდაჭერილი პროვაიდერები

- TBC  
- Credo  
- Bank of Georgia (BOG)  

---

## რას აკეთებს ეს პლაგინი

პლაგინი ქმნის სპეციალურ სამუშაო სივრცეს WooCommerce-ის ადმინისტრატორის პანელში, სადაც შეგიძლიათ:

- განვადების განაცხადის ხელით შექმნა  
- პროვაიდერის redirect ბმულის მიღება და გახსნა  
- განაცხადის სტატუსის შემოწმება  
- განაცხადების ისტორიის ნახვა  
- API მოთხოვნებისა და პასუხების ლოგების დათვალიერება  
- ტესტურ და რეალურ გარემოებს შორის გადართვა  
- ახალი პროვაიდერების დამატება გაფართოებადი არქიტექტურის საშუალებით  

---

## გამოყენების ძირითადი სცენარი

WI Installments Hub გათვლილია იმ შემთხვევებზე, როდესაც განვადების პროცესი სრულდება ადმინისტრატორის მხრიდან.

ეს განსაკუთრებით გამოსადეგია მაშინ, როცა:

- განაცხადები მუშავდება back-office გუნდის მიერ  
- კლიენტთან კომუნიკაციის შემდეგ იქმნება განაცხადი  
- საჭიროა რამდენიმე ბანკთან/პროვაიდერთან მუშაობა ერთ სისტემაში  

---

## ძირითადი ფუნქციები

- WooCommerce admin-ში ინტეგრირებული მართვის პანელი  
- სხვადასხვა პროვაიდერისთვის ცალკე სამუშაო გვერდები  
- ხელით განაცხადის შექმნის ფორმები  
- განაცხადების ისტორიის გვერდი  
- ლოგების ნახვის ინტერფეისი  
- მონაცემების შენახვა ბაზაში  
- ფაილური და ბაზაზე დაფუძნებული ლოგირება  
- სენსიტიური მონაცემების ნაწილობრივი დამალვა ლოგებში  
- ტესტური და production რეჟიმების მხარდაჭერა  
- გაფართოებადი არქიტექტურა ახალი პროვაიდერებისთვის  

---

## როგორ მუშაობს

სტანდარტული workflow:

1. პლაგინის პარამეტრებში შეიყვანეთ პროვაიდერის მონაცემები  
2. აირჩიეთ შესაბამისი პროვაიდერი ადმინისტრატორის პანელში  
3. შეავსეთ განაცხადის დეტალები (თანხა, პროდუქტი, მომხმარებელი და სხვა)  
4. შექმენით განაცხადი  
5. მიიღეთ redirect ბმული (საჭიროების შემთხვევაში)  
6. გადაამოწმეთ განაცხადის სტატუსი მოგვიანებით  
7. საჭიროების შემთხვევაში გამოიყენეთ ლოგები პრობლემის დასადგენად

- `wp_wi_installment_logs` — ლოგები
